Cinnamon - Portable edition

This is the standalone version of Cinnamon, which includes everything needed to run Cinnamon (both the server and the client) in one big zip archive.

You should use this version if you just want to try and see what Cinnamon has to offer. It is not intended for production purposes and has some technical limitations (due to the scripted environment - in all other regards, this version is feature complete).

Requirements

This version was tested on Windows Server 2008 64 Bit and Windows 10. It should run on all modern 64 Bit Windows systems - if you encounter problems, please contact us so we can improve your Cinnamon experience.

  • Note: if nothing works, check that you have cleared the downloaded archive's security flag _before_ unpacking.
  • You will need about 2 GByte of RAM.
  • Port 8080 on your machine must be available.
  • You must have .NET Framework 4 installed download for free from Microsoft
  • In "My Documents" create a folder 'cinnamon' with a sub folder 'checkout'. (The client expects this to exist, and the cinnamon.bat script will try to create this folder).

start the server

Double click on cinnamon.bat. Two windows will open, a command line window with some script output, and a java.exe window with incredible amounts of log messages. Even on a fast machine, this may take a minute or two.

start the client

  • Open the folder "client" and double click on "CinnamonDesktopClient.exe".
  • The repository list should show a green lighted Cinnamon server with a demo repository. Right-click on "demo" and log on as user admin with password 'admin'.

Login Dialog

  • Right-click on the admin user, choose "repository navigation" and you will see the repository window.

Repository View

Administration

Go to http://127.0.0.1:8080/cinnamon/admin/index to use the administration tool.

stop the server

After testing, stop the server by selecting the java window and pressing CTRL-C. It's possible that the process does not shut down cleanly - sorry, that seems to be the only annoying problem remaining... you may have to terminate the Java process via the task manager (or have a 500 MByte huge mean Java-process hanging around in the background, scaring other programs and blocking your 8080 port until shutdown).

stop the database

Select the cmd.exe window and press the enter key. The database should now shut down. If it takes too long, just close the window...
